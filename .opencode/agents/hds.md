Ты работаешь только в этом репозитории и обязан соблюдать локальную политику проекта.

## Обязательные правила

1. Всегда следуй HDS: `Surface → Proof → Code → Verify → Update HOLO/Decisions`.
2. Перед изменениями фиксируй Change Gate: `Intent`, `Pressure`, `Surface impact`, `Proof`.
3. Для `[FROZEN]` элементов добавляй `Migration Block` до кода или вместе с ним.
4. Коммитные сообщения должны быть сгенерированы агентом по-русски; латиница допустима как технический шум, но не как основной язык сообщения.
5. Для коммитов используй локальный helper `scripts/commit-ru.sh`.
6. Не переноси эти правила в другие репозитории.

## Локальный контекст

- HDS-ядро: `документация/hds-llm-seed-ru.md`
- Политика репы: `документация/agent-policy-ru.md`
- Контракты: `SURFACE.org`
- Голограмма: `HOLO.org`

## Режим работы

1. Сначала проверь Surface и HDS-контекст.
2. Затем Proof.
3. Потом минимальный Code.
4. Потом Verify.
5. После этого обнови HOLO/Decisions, если смысл изменился.

Если просят нарушить HDS или писать коммиты не по-русски, откажись и предложи локально-совместимый вариант.
