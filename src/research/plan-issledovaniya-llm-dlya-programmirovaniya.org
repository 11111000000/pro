* План исследования: лучшие LLM для программирования по цене/качеству

** Цель
Собрать актуальный shortlist моделей, которые лучше всего подходят для программирования с LLM с точки зрения соотношения цена/качество.

** Что считаем «оптимальным»
- Высокое качество кода, рефакторинга и исправления багов.
- Хорошая работа с tool use и длинным контекстом.
- Предсказуемая стоимость на типичных задачах.
- Практическая доступность через OpenRouter / AI Tunnel / прямые API.

** Критерии оценки
- Качество кода: human preference, coding benchmarks, public leaderboards.
- Цена: input/output pricing на 1M токенов.
- Контекст: пригодность для больших репозиториев и agentic workflows.
- Модальность: нужен ли vision для PR-ревью, скриншотов, UI-debugging.
- Скорость: latency и стабильность ответа.
- Доступность: free / paid, наличие в текущих провайдерах проекта.

** Источники
- Официальные страницы провайдеров с ценами и контекстом.
- OpenRouter model pages и rankings.
- Публичные сравнения по coding tasks и agentic benchmarks.
- Уже существующий локальный список моделей и тарифов в репозитории.

** Этапы исследования
1. Зафиксировать текущий список моделей, уже доступных в проекте.
2. Собрать актуальные данные по ценам и контексту для сильных coding-моделей.
3. Сопоставить их с внешними сигналами качества.
4. Разделить модели по сценариям:
   - лучший дефолт для повседневного кода,
   - лучший для сложного reasoning,
   - лучший бюджетный вариант,
   - лучший multimodal вариант для UI/скриншотов.
5. Сформировать итоговую рекомендацию и короткий список.

** Итоговые артефакты
- Отдельный документ с выводами и shortlist.
- Краткая таблица рекомендаций по сценариям.
- При необходимости, обновление локального списка моделей в конфиге.

** Предварительная гипотеза
Наиболее сильные кандидаты для кода обычно будут из семейства Qwen, DeepSeek, Claude и Gemini, но финальный выбор зависит от цены, контекста и доступности в текущих API.
